Implementing Low-Latency Traffic Steering with Route 53

Project Overview

This project focuses on configuring AWS Route 53 Latency Routing Policies. By setting up latency-based records, you ensure that end-users are automatically routed to the specific AWS endpoint (IP address) that offers the best response time based on their geographic location.
Prerequisites

->An active AWS Account.

->A registered Domain Name (or a private hosted zone setup).

->Two active endpoints (e.g., EC2 instances or Load Balancers) running in different AWS Regions
